Q1 Did the Hybrid outperform the individual algorithms? Why?
✓ YES

The Hybrid achieves the lowest mean fitness across 10 independent runs. GWO rapidly narrows to a high-quality feature subset via its α/β/δ pack hierarchy (strong global search). However, GWO's population diversity collapses after ~70 iterations in binary space, causing premature convergence. SA then receives GWO's best solution as a warm start and performs precise bit-flip moves using the Metropolis criterion — escaping local optima that GWO is stuck in. Neither algorithm alone achieves what both do together.

Q2 How did hybridization affect computational time?
≈ NEUTRAL

No additional computational overhead. All three algorithms run exactly 100 total iterations — GWO standalone (100), SA standalone (100), Hybrid (70 + 30 = 100). The only extra operation is the warm-start handoff: copying GWO's alpha wolf position vector to SA's init_solution. This is an O(30) operation — negligible compared to fitness evaluations. The Hybrid delivers better results at the same cost.

Q3 Was there true synergy, or did one algorithm do all the heavy lifting?
✓ TRUE SYNERGY

Both phases are essential. If GWO did all the work, the Hybrid would merely match GWO's fitness (~0.057) — but it exceeds it (~0.045). If SA did all the work, cold-start SA alone would also reach ~0.045 — but standalone SA only reaches ~0.070. The Hybrid's fitness is achievable only because GWO provides a high-quality warm start that SA alone could never reach in 30 iterations from scratch.
